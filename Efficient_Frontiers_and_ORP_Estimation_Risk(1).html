Efficient Frontier and Optimal Risky Portfolios: An Empirical and Simulation Study¶

This notebook explores the construction of efficient frontiers and optimal risky portfolios (ORPs) using stock return data. The analysis is carried out in three stages:

1. Empirical Efficient Frontiers (2010–2019)¶

  • Data is split into three windows: 2010–2013, 2014–2017, and 2018–2019.
  • For each period, we compute the efficient frontier using sample means and covariances, and identify the Optimal Risky Portfolio (maximum Sharpe ratio).
  • Portfolio weights, expected returns, volatilities, and Sharpe ratios are compared across periods.

2. Out-of-Sample Testing¶

  • The ORPs from 2010–2013 and 2014–2017 are evaluated using 2018–2019 returns.
  • This shows how efficient portfolios derived from historical data perform under new market conditions.
  • Results highlight that while earlier ORPs remain serviceable, they generally fall below the efficient frontier when tested out-of-sample.

3. Appendix: Estimation Risk Illustration¶

  • To illustrate why ORPs degrade out-of-sample, we simulate “noisy” efficient frontiers using synthetic data drawn from a known distribution.
  • Comparing these noisy frontiers to the true frontier demonstrates how estimation error inflates in-sample Sharpe ratios and pushes portfolios off the true frontier.
  • This reinforces the empirical findings: mean–variance optimization magnifies noise in expected returns and covariances, leading to portfolios that may look optimal historically but lose efficiency when conditions change.

def min_variance(cov, mu=None, desired_ret=None):

    n = cov.shape[0]

    def variance(w):
        return w @ cov @ w

    cons = [{'type': 'eq', 'fun': lambda w: np.sum(w) - 1.0}]
    if desired_ret is not None:
        if mu is None:
            raise ValueError("mu must be provided when desired_ret is specified.")
        cons.append({'type': 'eq', 'fun': lambda w: w @ mu - desired_ret})

    init_guess = np.ones(n) / n  # neutral start

    res = minimize(
        variance,
        init_guess,
        method="SLSQP",
        bounds=None,                 # shorting allowed
        constraints=cons,
        options={"ftol": 1e-12, "maxiter": 10000, "disp": False}
    )
    if not res.success:
        raise RuntimeError(f"Optimization failed: {res.message}")

    w = res.x
    return variance(w), w

def get_frontier(cov, mu, desired_rets, rf_daily):
    """
    Build efficient frontier by sweeping desired daily return targets.
    """

    rows = [min_variance(cov, mu, dr) for dr in desired_rets]

    df = pd.DataFrame(rows, columns=["Var_d", "Weights"])
    df["ER_d"] = desired_rets
    df["SD_d"] = np.sqrt(df["Var_d"])

    # Annualize
    df["ER_a"]  = 252 * df["ER_d"]
    df["Var_a"] = 252 * df["Var_d"]
    df["SD_a"]  = np.sqrt(252) * df["SD_d"]

    # Sharpe computed on an ANNUAL basis
    rf_a = 252 * rf_daily
    df["SR"] = (df["ER_a"] - rf_a) / df["SD_a"]

    return df

Conclusion¶

Across the three sample windows, the optimal risky portfolios highlight how changes in the return–covariance structure drive significant shifts in efficient allocations.

2010–2013 ORP delivered the highest Sharpe ratio (≈1.22), but required a short position in JPM.

2014–2017 ORP was more balanced and fully long, yet efficiency weakened with a lower Sharpe (≈1.08).

2018–2019 ORP produced the highest expected return (≈30%) at the cost of the highest volatility (≈24.6%). Its Sharpe (≈1.14) fell between the earlier windows, with allocations dominated by AAPL and NKE and a short in T, reflecting T’s weak risk–return contribution.

The out-of-sample test reinforces these findings: historical ORPs from 2010–2013 and 2014–2017 performed reasonably well in 2018–2019, landing close to the frontier but below the optimal set. This illustrates both the persistence and the limitations of in-sample optimality — portfolios built on past data may remain serviceable, but they rarely stay fully efficient when the market environment shifts.

Overall, the analysis demonstrates the importance of re-estimating mean–variance inputs over time and treating historical ORPs as guides rather than static prescriptions.

Appendix: Estimation Risk and Noisy Frontiers¶

While the analysis above focused on empirical data, it is important to recognize that mean–variance optimization is highly sensitive to estimation error. Even small differences in expected returns or covariances can shift the efficient frontier and the composition of the optimal portfolio. To illustrate this, we simulate returns from a known “true” distribution and compare the resulting noisy frontiers (based on finite samples) against the true frontier. This highlights how sampling noise can create large deviations and why portfolios that look optimal in-sample may perform less efficiently out-of-sample.

def random_frontier(mu_true, cov_true, rf,
                    n_pts=75, max_ann_ret=0.30, min_ann_ret=-0.05):

    sim_returns = np.random.multivariate_normal(mu_true, cov_true, size=100)

    mu_emp  = sim_returns.mean(axis=0)
    cov_emp = np.cov(sim_returns.T)

    lo_cap = min_ann_ret / 252.0
    hi_cap = max_ann_ret / 252.0

    lo = max(lo_cap, np.min(mu_emp) * 0.5)     # allow a bit negative
    hi = min(hi_cap, np.max(mu_emp) * 1.5)     # avoid crazy leverage

    # fallback if too narrow
    if hi <= lo:
        hi = lo + 1e-5

    sample_desired_rets = np.linspace(lo, hi, n_pts)

    return get_frontier(cov_emp, mu_emp, sample_desired_rets, rf)
